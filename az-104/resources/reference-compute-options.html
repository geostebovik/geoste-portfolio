Decision flow — which one do I reach for?
Need full OS control, custom drivers, or domain join? → VM. You're paying for infrastructure control — make sure you need it.
Building a web app or API that runs continuously? → App Service. Slots, managed TLS, auth, deployment center, auto-scale — all included.
Need to run something once, on a schedule, or for a short burst? → Container Instances. Spins up in seconds, bills by the second, gone when done.
Running many containers that talk to each other? → AKS. Orchestration, service discovery, rolling updates — Container Instances and App Service don't coordinate at that scale.
Packaging your App Service app as a container image? → Still App Service, just with container deployment mode. The platform is the same — the delivery mechanism changed.
